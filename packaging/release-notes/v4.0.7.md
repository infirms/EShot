## EShot v4.0.7

This release improves recording controls and expands experimental Linux desktop integration.

### Changes

- Made the GIF and video recording control bar draggable
- Prevented controls that overlap the selected region from appearing in Windows GDI recordings
- Improved full-screen and multi-monitor recording control placement on KDE Plasma 6
- Fixed incorrect recording borders and portal crop geometry on scaled or multi-monitor Wayland desktops
- Added persistent per-monitor portal source selection where supported
- Added dynamic recording shortcut guidance to the capture panel
- Added experimental GNOME Wayland capture, recording and global-shortcut integration
- Added PackageKit support with native package-manager fallbacks for optional Linux dependencies
- Improved AppImage integration, single-instance commands and desktop actions

Windows 10 and Windows 11 continue to use the existing FFmpeg GDI recording backend. Recording controls are hidden while they overlap the captured region and can be moved freely while they remain outside it.

KDE Plasma 6 Wayland remains the primary tested Linux target. GNOME Wayland support has received only limited testing, may be unstable or incomplete, and still needs feedback from different GNOME and portal versions. Other Linux desktop environments remain unsupported.
